# 哔哩哔哩关注列表批量管理工具

一个功能强大的哔哩哔哩关注列表管理工具，支持批量操作、数据可视化和智能分类。

## 🎯 项目概述

本工具旨在帮助B站用户更好地管理关注列表，提供智能分析、批量操作、数据可视化等功能，让关注列表管理变得更加高效。

## ✅ 已完成功能

### 🔍 关注列表管理
- ✅ **自动遍历关注列表**: 完整获取所有关注用户数据
- ✅ **数据同步**: 支持增量同步和强制刷新
- ✅ **本地数据存储**: SQLite数据库存储，支持离线查看
- ✅ **搜索筛选**: 支持按用户名、签名内容搜索
- ✅ **多种排序**: 按关注时间、用户名、等级等排序

### 🏷️ 智能分类系统
- ✅ **自动分类**: 基于用户名和签名的智能分类算法
- ✅ **10大分类**: 游戏、科技、知识、生活、娱乐、美食、时尚、汽车、财经、体育
- ✅ **手动调整**: 支持手动修改用户分类
- ✅ **分类统计**: 各分类用户数量和占比统计

### 📁 分组管理
- ✅ **分组同步**: 自动同步B站关注分组信息
- ✅ **分组查看**: 按分组查看关注用户
- ✅ **批量移动**: 支持批量移动用户到指定分组
- ✅ **分组统计**: 显示各分组用户数量

### ⚡ 批量操作
- ✅ **批量取消关注**: 支持选择多个用户批量取消关注
- ✅ **批量分组管理**: 批量移动用户到不同分组
- ✅ **选择模式**: 支持全选、反选、条件选择

### 📊 数据分析与可视化
- ✅ **分类分布图**: 饼图显示各分类占比
- ✅ **关注趋势**: 时间线显示关注增长趋势
- ✅ **用户统计**: VIP用户、认证用户等统计

### 💾 数据导入导出

- Python 3.8 或更高版本

### 快速开始

1. **克隆项目**
   ```bash
   git clone https://github.com/your-username/Bilibili-batch-operation-tool.git
   cd Bilibili-batch-operation-tool
   ```

   ```bash
   pip install -r requirements.txt
   ```

### 5. 数据分析

#### 活跃度评估原理 ✅
系统通过多维度指标综合计算UP主活跃度，帮助识别真正活跃的内容创作者：

**活跃度评估算法：** ✅ *已实现*
- **数据源**: 粉丝数量、视频总数、最后发布时间、总播放量等B站公开数据 ✅
- **时间因子**: 最后发布时间权重最高，7天内(0.9分)→30天内(0.7分)→90天内(0.5分)→更久(0.3分) ✅
- **内容产出**: 视频数量调节，100+视频(+0.1分)，少于10个视频(-0.1分) ✅
- **分数范围**: 0-1之间的标准化分数，便于比较和排序 ✅
- **默认策略**: 缺失数据时使用保守估值(0.6分)，避免误判为不活跃 ✅

**不活跃用户识别标准：** ✅ *已实现*
1. **主要标准**: 超过30天未发布新视频（基于最后发布时间）✅
2. **辅助标准**: 活跃度分数低于0.2 且 视频总数少于3个 ✅
3. **特殊情况**: 无视频活动记录且活跃度极低（<0.1）✅

**活跃度分级：** ⚠️ *部分实现*
- **高活跃(0.7-1.0)**: 近期有持续更新，内容丰富 ⚠️ *算法中未明确分级*
- **中活跃(0.4-0.7)**: 更新频率适中，有一定内容积累 ⚠️ *算法中未明确分级*
- **低活跃(0.2-0.4)**: 更新较少或间隔较长 ⚠️ *算法中未明确分级*
- **不活跃(0-0.2)**: 长期无更新或内容极少 ✅ *仅此级别在代码中有明确标准*

#### 数据可视化功能 ✅ *已实现*
- **分类分布图**: 饼图展示各分类占比情况 ✅
- **关注趋势图**: 时间线显示关注增长趋势 ✅
- **活跃度统计**: 活跃度分布和不活跃用户识别 ✅
- **智能洞察**: 基于数据的个性化分析结果 ✅

#### 智能洞察原则 ✅ *基本实现*
智能洞察功能采用多维度分析算法，为用户提供个性化的关注列表深度洞察：

**核心分析维度：** ✅ *已实现*
- **兴趣偏好识别**: 基于分类分布分析用户主要兴趣领域，识别最关注的内容类型 ✅
- **关注质量评估**: 结合UP主粉丝数、认证状态、活跃度等指标评估关注价值 ⚠️ *部分实现*
- **时间模式分析**: 分析关注时间规律和增长趋势，识别用户行为习惯变化 ✅
- **活跃度监控**: 实时监测不活跃用户占比，提供清理建议时机 ✅

**洞察生成逻辑：** ✅ *已实现*
- **兴趣聚焦**: 当某个分类占比超过20%时，识别为用户主要兴趣方向 ❌ *未明确实现此阈值*
- **异常检测**: 当不活跃用户占比超过30%时，发出清理建议警告 ✅
- **增长分析**: 基于最近3个月数据分析关注增长趋势（>10%快速增长，<-5%增长放缓）✅
- **个性化建议**: 根据用户当前关注分布和活跃度情况，提供定制化管理建议 ⚠️ *基础实现*

**数据安全保障：** ✅ *已实现*
- **本地计算**: 所有分析计算均在本地进行，不上传用户数据 ✅
- **实时更新**: 基于最新同步的关注列表数据进行动态分析 ✅
- **可选过滤**: 支持"忽略未分组"功能，专注分析已整理的用户数据 ✅
- **隐私保护**: 仅使用B站公开信息，不涉及用户隐私数据 ✅

#### 建议清理原则 ✅ *已实现*
系统采用多层级评估机制，为用户提供安全可靠的清理建议：

**不活跃用户识别机制：** ✅ *已实现*
1. **主要判断标准**: 超过30天未发布新视频（基于最后发布时间）✅
2. **辅助判断标准**: 活跃度分数低于0.2 且 视频总数少于3个 ✅
3. **特殊情况处理**: 无视频活动记录且活跃度极低（<0.1）✅
4. **综合评分排序**: 时间因子 + 活跃度因子 + 内容产出因子 ✅

**清理建议生成逻辑：** ✅ *已实现*
- **批量清理触发**: 当不活跃用户超过10个时，建议批量清理（最多显示50个优先级最高的）✅
- **分类整理建议**: 当"其他"分类用户超过20个时，建议进行智能重新分类 ✅
- **优先级排序**: 按不活跃程度综合评分，时间越久、活跃度越低、内容越少的排在前面 ✅

**数据安全策略：** ✅ *已实现*
- **保守估值**: 优先使用B站API真实数据，缺失时使用保守默认值避免误判 ✅
- **合理阈值**: 默认活跃度0.6分，避免将正常用户误判为不活跃 ✅
- **详细说明**: 为每个清理建议提供详细的不活跃原因说明 ✅
- **二次确认**: 支持手动确认机制，防止误操作 ✅

**清理建议分类：** ⚠️ *基础实现*
- **时间维度**: 长期无更新(30-90天)、极长期无更新(90天以上) ✅
- **活跃度维度**: 低活跃度创作者、僵尸账号检测 ✅
- **内容维度**: 内容产出极少、重复内容过多 ⚠️ *部分实现*
- **整理维度**: 未分类用户、分组混乱用户 ✅

**保护机制：** ⚠️ *部分实现*
- **白名单保护**: 自动保护认证用户、高粉丝数用户 ❌ *未明确实现*
- **批量限制**: 单次清理建议数量限制，避免过度清理 ✅
- **可撤销性**: 保留操作历史，支持后续数据恢复 ❌ *未实现*
- **用户控制**: 最终清理决定权完全由用户掌控 ✅

- ⚠️ **部分实现**: 核心功能已实现，但部分细节特性待完善
- ❌ **未实现**: 功能描述与实际代码不符，或功能尚未开发
- 📊 **理论估计值**: 基于算法的理论预期，未经大规模实际测试验证

### 实现状态总结

**✅ 完全实现的功能:**
- 自动分类算法：包括权重分配、关键词匹配、阈值控制等核心机制
- 活跃度评估：时间因子、内容产出调节、默认策略等
- 不活跃用户识别：30天标准、0.2阈值、特殊情况处理
- 清理建议生成：批量触发、分类整理、优先级排序
- 数据可视化：分类分布图、趋势图、统计图表
- 数据安全保障：本地计算、实时更新、隐私保护

**⚠️ 部分实现的功能:**
- 活跃度分级：仅"不活跃"级别有明确标准，其他级别待完善
- 关注质量评估：基础框架已有，复杂评估逻辑待增强
- 智能洞察：核心功能已实现，个性化程度有待提升
- 保护机制：基础保护已有，白名单和撤销功能待开发

**❌ 待实现的功能:**
- 兴趣聚焦20%阈值：洞察算法中未明确实现此特定阈值
- 白名单保护：认证用户和高粉丝用户的自动保护机制
- 可撤销性：清理操作的历史记录和恢复功能

本说明旨在为用户提供准确的功能预期，避免过度承诺。项目持续开发中，更多功能将逐步完善。
