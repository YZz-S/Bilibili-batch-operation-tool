# UP主统计数据功能说明

## 功能概述

本工具现在支持查看和管理UP主统计数据，包括查看保存的统计信息和在显眼位置提醒数据更新时间。

## 新增功能

### 1. UP主统计数据页面

**访问路径**: `/user-stats` 或通过导航栏"UP主统计数据"

**功能特点**:
- 📊 **数据概要**: 显示总用户数、有统计数据用户数、数据覆盖率等关键指标
- 🕒 **更新时间提醒**: 在页面顶部显眼位置显示数据最后更新时间
- 🔍 **数据筛选**: 可按有无统计数据、用户名、UID等条件筛选
- 📈 **排序功能**: 支持按更新时间、粉丝数、视频数、活跃度等字段排序
- 📄 **分页浏览**: 支持分页查看，每页50条数据
- 🎨 **美观界面**: 响应式设计，支持深色模式

**展示信息**:
- UP主基本信息（头像、用户名、分类）
- 粉丝数、视频数、总播放量
- 活跃度评分（高/中/低三个等级）
- 最后发布视频时间（天数 + 具体时间）
- 数据更新时间
- 数据状态（有数据/无数据）

### 2. 数据更新时间提醒

**显示位置**:
- 首页：统计卡片上方
- 数据分析页面：页面标题下方
- UP主统计数据页面：页面顶部

**提醒逻辑**:
- ✅ **当日更新**: 绿色背景，显示"统计数据已更新至今日"
- ℹ️ **3天内**: 蓝色背景，显示"统计数据最后更新于X天前"
- ⚠️ **4-7天**: 黄色背景，提示"统计数据已过期X天，建议重新同步"
- 🚨 **7天以上**: 红色背景，警告"统计数据严重过期X天，强烈建议立即同步"
- ❓ **无数据**: 黄色背景，提示"暂无统计数据，请先执行同步功能"

**快捷操作**:
- 📊 **查看详细数据**: 跳转到UP主统计数据页面
- 🔄 **同步数据**: 一键启动统计数据同步

### 3. API接口增强

新增两个API接口：

#### `/api/data/user-stats`
- **功能**: 获取UP主统计数据详情
- **参数**:
  - `page`: 页码（默认1）
  - `limit`: 每页数量（默认50）
  - `search`: 搜索关键词
  - `has_real_data`: 是否有真实数据（true/false）
  - `sort_by`: 排序字段
  - `sort_order`: 排序顺序（desc/asc）

#### `/api/data/user-stats/summary`
- **功能**: 获取UP主统计数据概要信息
- **返回**: 基本统计、时间信息、数据分布等

## 数据存储

### 数据库表结构

**user_stats表**:
```sql
CREATE TABLE user_stats (
    uid INTEGER PRIMARY KEY,              -- 用户ID
    fans_count INTEGER DEFAULT 0,         -- 粉丝数
    following_count INTEGER DEFAULT 0,    -- 关注数
    video_count INTEGER DEFAULT 0,        -- 视频数
    total_views INTEGER DEFAULT 0,        -- 总播放量
    last_video_time INTEGER DEFAULT 0,    -- 最后视频时间
    activity_score REAL DEFAULT 0.0,      -- 活跃度分数
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,  -- 更新时间
    FOREIGN KEY (uid) REFERENCES following_list (uid)
);
```

### 数据来源

- **真实数据**: 通过"同步真实数据"功能从B站API获取
- **模拟数据**: 通过"初始化用户统计数据"功能生成合理的模拟数据
- **数据更新**: 每次同步会更新或插入最新数据，保留历史记录

## 使用说明

### 首次使用

1. 如果是新安装，建议先执行"同步真实数据"获取准确信息
2. 如果只需要查看大概情况，可以使用"初始化用户统计数据"生成模拟数据

### 数据同步

1. **手动同步**: 在首页或数据分析页面点击"同步数据"按钮
2. **一键更新**: 使用"一键更新所有信息"功能，包含统计数据同步
3. **定期同步**: 建议每周同步一次以保持数据新鲜度

### 查看数据

1. **概览查看**: 在首页查看基本统计和更新时间提醒
2. **详细分析**: 在数据分析页面查看图表和深度分析
3. **数据浏览**: 在用户统计数据页面查看每个用户的详细信息

## 注意事项

### 数据准确性

- ✅ **真实数据**: 通过B站API获取的数据最准确
- 📊 **模拟数据**: 仅供参考，实际情况可能有差异
- 🕒 **时效性**: 数据不会自动更新，需要手动同步

### 性能考虑

- 🚀 **首次同步**: 大量用户的首次同步可能需要较长时间
- ⚡ **增量更新**: 后续同步会更新已有数据，速度较快
- 💾 **本地存储**: 所有数据存储在本地数据库，保护隐私

### 使用建议

- 📅 **定期同步**: 建议每周同步一次统计数据
- 🎯 **按需使用**: 根据实际需要选择同步范围
- 🔍 **数据筛选**: 使用筛选功能快速找到目标用户
- 📈 **趋势观察**: 结合时间维度观察用户活跃度变化

## 技术特性

- 🏗️ **响应式设计**: 适配各种屏幕尺寸
- 🌙 **深色模式**: 自动适配系统主题
- ⚡ **性能优化**: 分页加载，避免大量数据卡顿
- 🔒 **数据安全**: 本地存储，保护用户隐私
- 🎨 **美观界面**: 现代化UI设计，用户体验友好

## 故障排除

### 常见问题

**Q: 为什么显示"暂无统计数据"？**
A: 需要先执行"同步真实数据"或"初始化用户统计数据"功能。

**Q: 数据更新时间不对？**
A: 检查系统时间设置，或重新同步数据。

**Q: 页面加载很慢？**
A: 数据量大时可能较慢，建议使用筛选功能减少显示数据量。

**Q: 部分用户没有统计数据？**
A: 可能是API限制或用户隐私设置导致，属于正常现象。
