# B站风控处理指南

## 📖 概述

本指南帮助你理解并应对B站的风控机制，确保批量操作工具能够稳定运行。

## 🚨 常见风控错误码

| 错误码 | 含义 | 常见原因 | 处理建议 |
|--------|------|----------|----------|
| -352 | 频率限制 | API调用过于频繁 | 增加延迟，降低并发 |
| -503 | 服务暂时不可用 | 系统负载过高或触发保护 | 稍后重试，使用保守模式 |
| -799 | 请求过于频繁 | 短时间内大量请求 | 大幅增加延迟时间 |

## 🔍 风控触发的表现

### 从你的测试日志看到的典型表现：

1. **初期正常**：前两页数据正常获取
2. **开始限制**：第3页开始出现-352错误
3. **连续失败**：连续多次失败，延迟时间逐渐增加
4. **系统保护**：最终导致整个任务停止

### 其他常见表现：

- 响应时间异常增长
- 返回空数据或异常数据
- 需要验证码
- 临时封禁账号

## 🛡️ 应对策略

### 1. 立即解决方案（针对你的情况）

```bash
# 使用保守同步工具
python conservative_sync_resume.py 0 50

# 参数说明：
# 0: 从第一个用户开始
# 50: 只处理50个用户（小批量测试）
```

### 2. 保守模式特点

- **单线程处理**：避免并发请求
- **长延迟**：2-15秒延迟，根据情况自适应
- **风控检测**：实时监控风控状态
- **智能恢复**：风控触发后自动等待恢复
- **缓存机制**：避免重复请求

## 📊 使用建议

### 根据用户数量选择策略：

| 用户数量 | 推荐模式 | 预计时间 | 说明 |
|----------|----------|----------|------|
| 1-50 | 保守模式 | 5-15分钟 | 测试验证 |
| 50-200 | 保守模式 | 30-60分钟 | 小批量处理 |
| 200-500 | 分批保守模式 | 2-4小时 | 分多次处理 |
| 500+ | 分批保守模式 | 分多天处理 | 每天处理一部分 |

### 对于你的1300用户情况：

1. **先用保守模式处理50个用户测试**
2. **如果成功，逐步增加到100、200**
3. **分批处理，每批200-300用户**
4. **每批之间间隔2-4小时**

## 🔧 实际操作步骤

### 步骤1：测试保守模式

```bash
# 测试前50个用户
python conservative_sync_resume.py 0 50 --max-failures 5

# 检查结果，如果成功率>80%，继续下一步
```

### 步骤2：小批量处理

```bash
# 处理第1批（用户1-200）
python conservative_sync_resume.py 0 200

# 等待2小时后处理第2批（用户201-400）
python conservative_sync_resume.py 200 200

# 以此类推...
```

### 步骤3：监控和调整

- 监控成功率，如果低于70%，增加延迟
- 观察日志中的风控警告
- 根据实际情况调整批次大小

## ⏰ 时间预估

### 保守模式时间计算：

- **单用户处理时间**：3-8秒（包括延迟）
- **50用户**：大约5-15分钟
- **200用户**：大约30-60分钟
- **1300用户**：分批处理，总计6-12小时（分多天）

### 优化建议：

1. **夜间处理**：晚上网络负载较低
2. **分散处理**：不要连续长时间运行
3. **缓存利用**：重复运行时利用缓存

## 🔍 故障排除

### 如果仍然触发风控：

1. **增加延迟**：
   ```bash
   # 修改配置，将base_delay增加到5.0
   ```

2. **减少批次**：
   ```bash
   # 每次只处理20-30个用户
   python conservative_sync_resume.py 0 30
   ```

3. **更换时间段**：
   - 避开高峰期（晚上7-11点）
   - 选择凌晨或早晨时段

4. **检查网络**：
   - 确保网络稳定
   - 避免使用代理或VPN

### 如果数据不准确：

1. **清除缓存**：删除临时缓存文件
2. **检查Cookie**：确保Cookie仍然有效
3. **验证账号**：登录B站网页确认账号正常

## 🆘 紧急情况

如果账号被临时限制：
1. **停止所有操作**
2. **等待24小时**
3. **登录网页确认恢复**
4. **使用更保守的参数重新开始**
