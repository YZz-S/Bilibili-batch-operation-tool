# 一键更新功能说明

## 功能概述

新增的一键更新功能可以自动执行多个维护任务，帮助用户快速更新关注列表的所有相关信息。

## 功能包含的更新内容

### 1. 同步关注列表
- 从B站API获取最新的关注用户列表
- 更新用户基本信息（昵称、头像、签名等）
- 限制处理50个用户（避免执行时间过长）

### 2. 更新分组信息
- 同步B站的关注分组结构
- 更新分组名称和用户数量
- 确保本地数据库与B站保持一致

### 3. 同步用户统计数据
- 获取用户最新的粉丝数、关注数、视频数
- 更新最后发布视频时间
- 计算用户活跃度分数
- 限制处理20个用户（避免API频率限制）

### 4. 修复等级信息
- 获取用户的真实等级信息
- 修复数据库中等级为0的记录
- 限制处理30个用户

### 5. 智能自动分类
- 对未分类的用户进行智能分类
- 基于用户名、签名等信息识别类别
- 支持游戏、科技、知识、生活等12个分类

## 使用方法

### 在首页
1. 点击主要操作区的 **"一键更新所有信息"** 按钮（火箭图标）
2. 或在快速操作卡片中点击相同按钮

### 在关注列表页面
- 点击页面右上角的 **"一键更新所有信息"** 按钮

### 在数据分析页面
- 点击页面右上角的 **"一键更新所有信息"** 按钮

## 执行过程

1. **确认对话框**：显示将要执行的操作列表
2. **后台执行**：任务在后台异步执行，不会阻塞页面使用
3. **进度反馈**：
   - 首页：显示详细的进度模态框
   - 其他页面：定时刷新数据并显示进度提示
4. **完成通知**：任务完成后显示结果通知

## 执行时间

- **预计时间**：3-10分钟（取决于网络状况和API响应速度）
- **并发限制**：为避免触发B站API频率限制，各步骤有适当的延迟
- **自动重试**：遇到API限制时会自动调整请求频率

## 注意事项

1. **网络要求**：需要稳定的网络连接
2. **Cookie配置**：必须在设置页面配置有效的B站Cookie
3. **耐心等待**：整个过程需要几分钟时间，请不要关闭页面
4. **数据安全**：所有操作都基于B站官方API，安全可靠
5. **频率限制**：如遇到API限制，系统会自动调整，这是正常现象

## API端点

新增的API端点：`POST /api/bilibili/one-click-update`

可选参数：
- `sync_limit`: 同步关注列表的用户数量限制（默认50）
- `stats_limit`: 同步统计数据的用户数量限制（默认20）

## 技术实现

- **后台任务**：使用FastAPI的BackgroundTasks实现异步执行
- **错误处理**：完善的异常处理和日志记录
- **进度追踪**：详细的日志记录和状态反馈
- **资源限制**：合理的数量限制和延迟控制
